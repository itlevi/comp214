3.1 Describe Graphical User Interfaces – Windows

I am primarily use Apple and Linux for my personal computing needs, but I do work and support a Windows environment as an IT technician, so I do have quite a bit of hands on experience with most Windows GUIs since Windows 3.1.

I will focus on the Windows 10 GUI as it is the most recent version of Windows. As I spend more time working with Windows, I find myself using the command-line interface (CLI) more often that the GUI, and the reason for that is the Windows 10 GUI is a user experience nightmare.

Everything in the GUI is all over the place and there are too many ways to complete a single task. One example is the new Windows Settings GUI introduced in Windows 8.0 which was intended to replace the traditional Control Panel pane. Both still exist and it is completely unnecessary to have both. Please Windows, just pick one already.

The start menu in Windows 10 is a kitchen sink of different options, all competing for the user’s attention. You have 3 columns of different styled icons and layouts. The far-left column is an outlined set of textless, mono-coloured icons, the second column is a scrollable list of icons and text with arbitrarily picked colour backgrounds, and the largest of the columns is the grid pane layout remnant from Windows 8.0. It is a lot of information to take in and I have personally witnessed many non-technical users become overwhelmed while trying to navigate the menagerie of different options.

Thankfully the search function is visible by default and is really the only part of the Windows UI that I use frequently. It is in my opinion the fastest way to navigate to common tools like Control Panel, Device Manager, Command Prompt and even files. When I train users on how to use Windows, I tend to steer them into making sure the Search function becomes a habit.

I don’t think Windows 10 is all bad. It is still a huge improvement over Windows 8.0/8.1, and I look forward to seeing their next iteration of GUIs.
